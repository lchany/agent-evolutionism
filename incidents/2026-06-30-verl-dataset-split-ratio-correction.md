# VERL dataset split ratio correction

## Trigger Signal

User corrects a generated VERL dataset because final train/test counts do not follow the expected standard training/testing ratio.

## Context

Task: generate a KITTI tracking based VERL multi-image dataset where one sample contains 28 physical images, organized as 7 logical groups with 4 frames per group.

Dataset output path: `/mnt/disk2t/l30002999/dataset/kitti_28f_7x4_verl/`.

Builder script: `/mnt/disk2t/l30002999/tmp/kitti_28frames_dataset/make_kitti_28f_7x4_verl.py`.

## Failed Command Or Operation

Initial dataset generation used KITTI source directory names/counts directly: source `training` became final `train.parquet`, source `testing` became final `test.parquet`.

## Error Signature

Final split was 72 train rows and 118 test rows, so the training split was smaller than the test split and did not match the expected standard ratio.

## Failed Attempts

- Generated a VERL-compatible parquet schema and validated image/group structure.
- Missed the split-ratio requirement by preserving raw source split counts.

## Root Cause

The builder treated source dataset directory names as final ML split semantics. For generated VERL training datasets, source folders are only input pools; they must not silently determine final train/test ratio.

## Resolution

Regenerated by merging the KITTI source `training` and `testing` sample pools first, then applying a deterministic standard 80/20 train/test split unless the user specifies another ratio.

Also recorded an active global rule and project memory correction: do not let source directory names or raw source split counts determine final VERL train/test ratio.

## Verification

Verified inside container `verl_lazy_clean_baseline`:

- `train.parquet`: 152 rows
- `test.parquet`: 38 rows
- train ratio: `152 / (152 + 38) = 0.8`
- each sample has 28 images
- each sample has 7 logical groups
- prompt contains 28 `<image>` placeholders
- per-group image shapes and mem values match the user-provided specification

## Reuse Notes

For future VERL dataset generation:

1. Ask or infer the intended split ratio.
2. If unspecified, default to 80/20.
3. Treat raw source directories as input pools, not final split semantics.
4. Record the split strategy in `summary.json` and `README.md`.
5. Verify final counts and ratio in the same container/environment where training will consume the dataset.

## Non-Applicable Cases

Do not apply this rule when the user explicitly requests preserving an upstream benchmark split or source-defined train/test split.
