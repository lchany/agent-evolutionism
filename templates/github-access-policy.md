# GitHub Access Policy

The `experience-vault` repository must be private.

Recommended rules:

1. Grant write access only to trusted maintainers.
2. Use SSH keys or GitHub fine-grained tokens; do not store credentials in the vault.
3. Require manual review before pushing generated records.
4. Keep raw logs and credentials out of Git history.
5. Rotate credentials if sensitive data is accidentally committed.
6. Prefer small, focused commits with clear messages.
